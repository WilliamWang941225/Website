# William Website - React + TypeScript version

This is a Vite + React + TypeScript conversion of the original static HTML website.

## What was added

* The website is split into TSX components in `src/components/`.
* The `My Resources` section is generated from data instead of repeated HTML.
* Resource dates are formatted as `YYYY/MM/DD`.
* The resource list is sorted from latest to oldest in `src/components/ResourceList.tsx`.
* A build-time script detects local file modified dates from files in `public/document/`.

## How to run

Install Node.js first. Then run:

```bash
npm.cmd install
npm.cmd run dev
```

To build the final static website:

```bash
npm.cmd run build
```

## How resource dates work

Edit resource information here:

```text
scripts/resourcesBase.mjs
```

Then run:

```bash
npm.cmd run generate-resources
```

The script generates:

```text
src/data/resources.ts
```

The generated file should not be edited directly.

For each resource, the script checks `dateSourceHref`, for example:

```js
dateSourceHref: "/document/Fluid Dynamics.pdf"
```

It then looks for the actual file at:

```text
public/document/Fluid Dynamics.pdf
```

If the file exists, it uses the file modified date and formats it as:

```text
YYYY/MM/DD
```

If the file is missing, the date becomes empty unless you provide `fallbackDate`.

## Important note

The uploaded zip used here did not include the files inside `document/`, so most automatically detected dates are currently empty. Put your PDF/doc files into:

```text
public/document/
```

Then run:

```bash
npm.cmd run generate-resources
```

The dates will be detected automatically.

## Website update workflow

Use this workflow whenever you revise the website content.

### 1. Edit the website locally

For resource entries, edit:

```text
scripts/resourcesBase.mjs
```

This is where you should change resource titles, descriptions, links, `dateSourceHref`, and `fallbackDate`.

Do not directly edit:

```text
src/data/resources.ts
```

because this file is automatically generated and will be overwritten when you run the resource generation command.

For page structure or text outside the resource list, edit files such as:

```text
src/App.tsx
src/components/
```

For CSS styling, edit:

```text
src/styles.css
```

For PDF files and other documents, put them in:

```text
public/document/
```

### 2. Regenerate resource data

After editing `scripts/resourcesBase.mjs` or adding files to `public/document/`, run:

```bash
npm.cmd run generate-resources
```

This updates:

```text
src/data/resources.ts
```

with the latest resource information and detected file dates.

### 3. Preview the website locally

Run:

```bash
npm.cmd run dev
```

Open the local Vite address shown in the terminal, usually:

```text
http://localhost:5173/
```

Check that the website looks correct before publishing.

### 4. Build the website

After confirming that the local preview is correct, stop the dev server with:

```text
Ctrl + C
```

Then run:

```bash
npm.cmd run build
```

This creates the final static website files in:

```text
dist/
```

### 5. Deploy the website

If the project uses the `gh-pages` deployment method, run:

```bash
npm.cmd run deploy
```

This publishes the built `dist/` folder to the online GitHub Pages website.

### 6. Commit and push the source code

After deployment, save the source code changes to GitHub:

```bash
git add .
git commit -m "Update website content"
git push
```

A typical full update sequence is:

```bash
npm.cmd run generate-resources
npm.cmd run dev
```

After checking the local website:

```bash
Ctrl + C
npm.cmd run build
npm.cmd run deploy
git add .
git commit -m "Update website content"
git push
```
